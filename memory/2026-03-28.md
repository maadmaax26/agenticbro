| # | Red Flag | Weight | Example |
|---|----------|--------|---------|
| 1 | Unverified Page | +5 | No blue checkmark badge |
| 2 | New Page | +4 | Created < 30 days ago |
| 3 | Guaranteed Returns | +9 | "Risk-free", "100% returns" |
| 4 | VIP Tier Structure | +7 | "VIP Tier 1: $100/month" |

| 7 | Urgency Tactics | +6 | "Limited time only!" |
| 8 | Requests Crypto | +10 | "Send to wallet address" |

### Risk Scoring Formula
```
Risk Score = (Sum of red flag weights / 70) × 10
```

### Risk Levels
- **0-3:** LOW - Verify, but generally safe
- **3-5:** MEDIUM - Exercise caution, verify claims
- **5-7:** HIGH - Avoid interaction, warn others
- **7-10:** CRITICAL - Do not interact, report immediately

### Why It's Not Working
The routing logic in `/agents/main/AGENT.md` is **instructions for the agent**, not code that automatically runs. The main agent reads the instructions but doesn't automatically delegate based on them.

### What's Needed
To actually route messages through local models:
1. Implement routing logic as actual code (not just documentation)
2. Have main agent analyze requests and use `sessions_send` to delegate
3. Ensure specialized agents are running and ready
